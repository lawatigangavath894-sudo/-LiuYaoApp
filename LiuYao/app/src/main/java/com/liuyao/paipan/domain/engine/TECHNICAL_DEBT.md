# 引擎层技术债登记

集中追踪 `domain/engine` 的已知精度限制与待办,按优先级排序。
代码中对应位置保留 `TODO` 注释,此处统一登记优先级与影响面。

## P1(影响排盘正确性,优先处理)

- **节气定月精度**(CalendarCalculator):当前用低精度节气公式,临近节气交接当天月柱可能 ±1 日误差。
  影响:月破/月建判断在临界日可能错。建议接入高精度节气表或天文算法。
- **晚子时分日**(CalendarCalculator):未处理晚子时(23:00–24:00)是否进位日柱的流派差异。
  影响:晚子时起卦的日柱、旬空、日辰生克可能偏移一日。

## P2(影响细节,可逐步完善)

- **爻间关系完整化**(ChartStatusCalculator):爻间两两刑冲合害、三合局/三会局、暗动判定尚未完整。
  影响:`LineStatus` 的 isCombined/isClashed 等只覆盖部分情形,匹配器相关条件可能漏判。
- **年柱校验**(CalendarCalculator):立春换年的边界需与权威万年历逐年校验。

## P3(流派差异,记录备查)

- **伏神取用流派**(HiddenFlyingSpiritCalculator):个别流派对"伏神是否以首卦为准/是否取月卦身"有差异,本版采用最通行做法。
- **真太阳时校正**(CalendarCalculator):未做经度/均时差校正,按输入本地钟表时间。多数日常占卜可接受。

## 处理建议

P1 项建议在"排盘算法精度"专项里集中攻关(需引入节气数据源);P2 在断语匹配深化时一并补;P3 作为可配置项(设置页"流派"开关)在远期考虑。
